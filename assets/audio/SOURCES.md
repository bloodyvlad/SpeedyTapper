# Audio sources

- `fluorescent-hum.wav` is an original sample-seamless PCM loop created for SpeedyTapper from 60, 120, 240, and 360 Hz sine components.
- `oops.wav` is an original 48 kHz/16-bit PCM descending failure cue generated by `scripts/generate-oops.py`. Its waveform has exact-zero endpoints and raised-cosine attack/release ramps.
- `neon-circuit-refined.m4a`, `deep-current.m4a`, and `power-grid.m4a` are original SpeedyTapper procedural compositions with no third-party samples or royalty requirements. Each runtime file is 48 kHz stereo AAC-LC with a 192 kbps target and contains four regions: 100 BPM menu/1x1 at frame 0, 120 BPM 2x2 at frame 460,800, 140 BPM mature 4x4 at frame 844,800, and 168 BPM endurance at frame 1,173,943, ending at Apple/CoreAudio valid frame 1,448,208.
- The corresponding 48 kHz/16-bit stereo rollback masters are `music-masters/neon-circuit-refined.wav`, `music-masters/deep-current.wav`, and `music-masters/power-grid.wav`. `scripts/generate-production-music.py` reproduces the masters and runtime AACs without lossy transcoding and validates the decoded AAC boundaries.
- `interactive-neon-circuit-refined.m4a`, `interactive-deep-current.m4a`, and `interactive-power-grid.m4a` are original backing-only Interactive Music sprites derived procedurally from the same SpeedyTapper synthesis voices, with no third-party samples or royalty requirements. Each 48 kHz stereo AAC-LC file has a 192 kbps target and contains twelve guard-isolated, four-bar backing loops from 100 through 168 BPM plus authored bridge beats in both directions between adjacent states. The backing uses modal pedal harmony so tap-triggered motif notes remain compatible regardless of player timing.
- The corresponding 48 kHz/16-bit stereo PCM rollback masters are `interactive-music-masters/neon-circuit-refined.wav`, `interactive-music-masters/deep-current.wav`, and `interactive-music-masters/power-grid.wav`. Exact decoded-frame cue points, transition boundaries, and CoreAudio valid-frame counts are retained in `interactive-music-masters/manifest.json`.
- `interactive-notes-neon-circuit-refined.wav`, `interactive-notes-deep-current.wav`, and `interactive-notes-power-grid.wav` are original 48 kHz/16-bit mono PCM runtime note banks and lossless masters. Each contains five ordered modal degrees plus the repeated octave used by that track's fixed 16-note tap motif in half-second, exact-zero-ended slots. The motif is the canonical contour extracted from the approved stage-two lead; player color and reaction time never select its pitch. At runtime, the controller raises motif degrees according to the same authored richness state as the backing, wrapping only at the stored octave and using at most 2× Web Audio playback for the next octave. This reuses the approved lossless bank without replacing or re-encoding it.
- `scripts/generate-interactive-music.py` reproduces the Interactive backing masters, runtime AAC sprites, PCM note banks, and manifest. It validates AAC-LC format, decoded duration and discontinuities, loop edges after AAC decoding, CoreAudio valid frames when available, and every PCM note boundary. The approved legacy music assets and generator are not modified by this process.
- `neon-circuit-v1.m4a` is the retained prior runtime soundtrack. Its lossless rollback master is `music-previews/01-neon-circuit-clicksafe-v4.wav`; `scripts/generate-neon-hifi-preview.py` reproduces both copies. Do not delete this prior version until the rotating production set is physically validated and has a separate recoverable release.
